The goal is to create a simple yet effective prototype to demonstrate the team’s technical capabilities to potential investors.

Project Workflow

  1. Provisioning the Server
    • Use AWS EC2 to create an Ubuntu Linux instance.
    • Configure SSH access for secure server management.
  2. Installing the Web Server
    • Install Apache (or Nginx) web server using package management tools like apt
    • Start and enable the web server to ensure it runs on boot.
  3. Creating and Deploying the HTML Page
    • Develop a simple HTML file (index.html) containing project details and the developer's bio.
    • Place the file in the appropriate web server directory (e.g., /var/www/html for Apache).
  4. Networking Configuration
    • Update firewall settings (using tools like ufw) to allow incoming HTTP traffic on port 80
    • Verify that the server is accessible via the public IP address or domain name.
  5. Documentation and GitHub Submission
    • Write clear, step-by-step documentation in the README.md file.
    • Include the public IP address or URL, instructions to replicate the setup, and additional useful information.
    • Commit the HTML file, screenshots, and documentation to a GitHub repository for submission.
